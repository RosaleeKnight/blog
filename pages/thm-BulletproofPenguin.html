TryHackMe Bulletproof Pinguin | Oct 30, 2023

This CTF I completed on Haloween while still working on increasing my position on the TryHackMe October Leaderboard for Canada. At time of writing I'm in 7th place and have succeeded in my goal on getting on the board for the month. This room is described as Linux Hardening difficulty Easy. You can try out the room here.

Start the machine and SHH in with thm@(the machine IP) with the password p3ngu1n.

---Task 1 Introduction---

Question: Complete above no answer needed.

---Task 2 Redis Server---

Question: What is the redis_nopass flag?

Steps: Use sudo nano/etc/redis/redis.conf command and uncomment requirepass and change foobared to any password and use sudo systemctl restart redis to restart server and run get-flags command for flag.

Answer: THM{ae4e5bb7aac2c2252363ca466f10ffd0}

---Task 3 Report Default Community Names of SNMP Agent---

Question: What is the snmp_public flag?

Steps: Use sudo Nano/etc/snmpd.conf and change rocummunity public to rocummunity any password and run get-flags for the flag.

Answer: THM{aa397a808d527fd71f023c78d3c04591}

---Task 4 Ngnix running as root---

Question: What is the nginx_asroot flag?

Steps: Use sudo Nano /etc/ngnix/nginx.conf and change user root to user www-data www-data; and use sudo service nginx restart to restart and use get-flags for the flag.

Answer: THM{bebb02b22bb56b2f79ba706975714ee2}

---Task 5 Cleartext Protocols---

Question: What other cleartext service is running on port 69/udp?

Steps: Use sudo apt remove telnetd -y to remove Telnet and use sudo lsof -I :69 and look under name section.

Answer: TFTP

Question: What is the cleartext_services flag?

Steps: Use sudo service inetd stop to stop service and use get-flag for flag.

Answer: THM{33704d74ec53c8cf50daf817bea836a1}

---Task 6 Weak SSH Cryto---

Question: What is the ssh_weak_macs flag?

Steps: Use sudo nano etc/shh/sshd_config and remove hmac-md5-96 and use sudo service sshd restart to restart service and use get-flag for flag.

Answer: THM{e3d6b82f291b64f95213583dcd89b659}

Question: What is the ssh_weak_kex flag?

Steps: Use sudo nano etc/shh/sshd_config and remove diffie-hellman-group1-sha1 and use sudo service sshd restart to restart service and use get-flag for flag.

Answer: THM{d9baf598ee934d79346f425a81bd693a}

Question: What is the ssh_weak_ciphers flag?

Steps: Use sudo nano etc/shh/sshd_config and remove 3des-cbc, aes128-cbc, aes256-cbc and and use sudo service sshd restart to restart service and use get-flag for flag.

Answer: THM{9ff9c182cad601291d45951c01d0b2c7}

---Task 7 Anonymous FTP Login Reporting---

Question: What is the anon_ftp flag?

Steps: Use sudo nano /etc/vsftpd.conf and change anonymous_enable from yes to no and use sudo systemtl restart vsftpd to restart and get-flag to get flag.

Answer: THM{f20b5ff5a3d4c779e99c3a93d1f68c6d}

---Task 8 Weak Passwords---

Question: What is the change_pass flag?

Steps: Use sudo passwd mary and give it a random password and use sudo passwd munra and give it a random password and use get-flags to get flag.

Answer: THM{be74a521c3982298d2e9b0e347a3807d}

Question: What is the unused_accounts flag?

Steps: Use sudo userdel Joseph and and use sudo userdel test1 and use get-flags to get flag.

Answer: THM{1b354db0e71f75057abe69de26a637ab}

---Task 9 Review Sudo Permissions---

Question: What is the sudoers_munra flag?

Steps: Use sudo visudo and remove munra and use get-flags to get flag.

Answer: THM{1e9ee13fb42fea2a9eb2730c51448241}

Question: What is the sudoers_mary flag?

Steps: Use sudo visudo and add mary ALL=(ALL) NOPASSWD: /user/bin/ss and use get-flags to get flag.

Answer: THM{a0bcb9b72fd26d0ad55cdcdcd21698f1}

---Task 10 Exposed Database Ports---

Question: What is the mysql_port_public flag?

Steps: Use sudo nano /etc/mysql/mysql.conf.d/MySQL.cnf and change bind-address = 0.0.0.0 to bind-address = 127.0.0.1 and use sudo service MySQL restart and use get-flags to get flag.

Answer: THM{526e33142b54e13bb47b17056823ab60}

Question: What is the redis_port_public flag?

Steps: Use sudo nano /etc/redis/redis.conf and and use sudo service redis restart to restart and use get-flags to get flag.

Answer: THM{20a809866dbcf94109189c5bafabc5c2}
